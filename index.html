xmltool’s documentation!¶

Introduction¶

xmltool is a python package to manipulate XML files. It’s very useful to update some XML files with the python syntax without using the DOM. The main goal of this package was to create a HTML form to edit and create a XML file. The form generation is based on a dtd file. You can find the source on github.

Basic¶

Creating a XML¶

General case¶

You will see in this example how it’s easy to create an XML file according to the following DTD file:

<!ELEMENT movie (title, realisator, characters)>
<!ATTLIST movie idmovie ID #IMPLIED>
<!ELEMENT title (#PCDATA)>
<!ELEMENT realisator (#PCDATA)>
<!ELEMENT characters (character+)>
<!ELEMENT character (#PCDATA)>
<!ATTLIST character idcharacter ID #IMPLIED>
>>> from lxml import etree
>>> import xmltool
>>> from xmltool import dtd_parser
>>> dtd_url = 'http://xml-tools.lereskp.fr/examples/movie.dtd'
>>> dic = dtd_parser.parse(dtd_url=dtd_url)
>>> movie = dic['movie']()
>>> title = movie.add('title', 'Movie title')
>>> print title._value
Movie title
>>> realisator = movie.add('realisator', 'realisator name')
>>> characters = movie.add('characters')
>>> c1 = characters.add('character', 'character 1')
>>> c2 = characters.add('character', 'character 2')
>>> print etree.tostring(movie.to_xml(), pretty_print=True)
<movie>
  <title>Movie title</title>
  <realisator>realisator name</realisator>
  <characters>
    <character>character 1</character>
    <character>character 2</character>
  </characters>
</movie>

Case with conditional elements¶

Now we will see how to handle case with some conditional elements. It’s ‘(qcm|mqm)*’ in the following DTD files:

<!ELEMENT Exercise (question, (qcm|mqm), comments*)>
<!ELEMENT question (#PCDATA)>
<!ELEMENT qcm (good-choice, bad-choice*)>
<!ELEMENT mqm (good-choice, bad-choice*)>
<!ELEMENT comments (good-comment, bad-comment)+>
<!ELEMENT good-choice (#PCDATA)>
<!ELEMENT bad-choice (#PCDATA)>
<!ELEMENT good-comment (#PCDATA)>
<!ELEMENT bad-comment (#PCDATA)>
>>> import xmltool
>>> from xmltool import dtd_parser
>>> dtd_url = 'http://xml-tools.lereskp.fr/examples/exercise.dtd'
>>> dic = dtd_parser.parse(dtd_url=dtd_url)
>>> exercise = dic['Exercise']()
>>> question = exercise.add('question', 'the question')
>>> qcm = exercise.add('qcm')
>>> # Now we can't add a mqm since a qcm exists
>>> exercise.add('mqm')
Traceback (most recent call last):
Exception: qcm already defined
>>> exercise.add('qcm')
Traceback (most recent call last):
Exception: qcm already defined
>>> c1 = qcm.add('good-choice', 'choice 1')
>>> print etree.tostring(exercise.to_xml(), pretty_print=True)
<Exercise>
  <question>the question</question>
  <qcm>
    <good-choice>choice 1</good-choice>
  </qcm>
</Exercise>

Accessing to the property¶

There is two ways to access to a property:
  • object style: obj.my_prop
  • dict style: obj[‘my_prop’]. The dict style is very useful to access to the properties containing a dash.

See the example below:

>>> from lxml import etree
>>> import xmltool
>>> from xmltool import dtd_parser
>>> dtd_url = 'http://xml-tools.lereskp.fr/examples/exercise.dtd'
>>> dic = dtd_parser.parse(dtd_url=dtd_url)
>>> exercise = dic['Exercise']()
>>> question = exercise.add('question', 'the question')
>>> comments = exercise.add('comments')
>>> c1 = comments.add('good-comment', 'comment 1')
>>> print exercise.question._value
the question
>>> exercise.comments == exercise['comments']
True
>>> exercise['comments'][0]['good-comment']._value
'comment 1'

Loading a XML file¶

For this example we will load the following XML file:

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie PUBLIC "http://xml-tools.lereskp.fr/examples/movie.dtd" "http://xml-tools.lereskp.fr/examples/movie.dtd">
<movie idmovie="M1">
 <title>Full Metal Jacket</title>
 <realisator>Stanley Kubrick</realisator>
 <characters>
  <character idcharacter="C1">Matthew Modine</character>
  <character idcharacter="C2">Vincent D'Onofrio</character>
 </characters>
</movie>
>>> xml_filename = 'examples/movie.xml'
>>> obj = xmltool.load(xml_filename)
>>> print obj._attributes
{'idmovie': 'M1'}
>>> print obj 
<xmltool.dtd_parser.movie object at 0x...>
>>> print obj.title._value
Full Metal Jacket
>>> print obj.characters.character[0]._value
Matthew Modine
>>> print obj.characters.character[0]._attributes
{'idcharacter': 'C1'}

Writing a XML file¶

>>> xml_filename = 'examples/movie.xml'
>>> obj = xmltool.load(xml_filename)
>>> obj.title._value = 'My new title'
>>> obj.write('examples/movie-updated.xml')

See the content of the new XML file:

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie SYSTEM "http://xml-tools.lereskp.fr/examples/movie.dtd">
<movie idmovie="M1">
  <title>My new title</title>
  <realisator>Stanley Kubrick</realisator>
  <characters>
    <character idcharacter="C1">Matthew Modine</character>
    <character idcharacter="C2">Vincent D'Onofrio</character>
  </characters>
</movie>

Updating a XML file after a dtd change¶

Our dtd file:

<!ELEMENT movie (title, realisator, characters)>
<!ATTLIST movie idmovie ID #IMPLIED>
<!ELEMENT title (#PCDATA)>
<!ELEMENT realisator (#PCDATA)>
<!ELEMENT characters (character+)>
<!ELEMENT character (#PCDATA)>
<!ATTLIST character idcharacter ID #IMPLIED>

Now we add a comment field required. In XML the tag is required, not the content!

<!ELEMENT movie (title, realisator, characters, comment)>
<!ATTLIST movie idmovie ID #IMPLIED>
<!ELEMENT title (#PCDATA)>
<!ELEMENT realisator (#PCDATA)>
<!ELEMENT characters (character+)>
<!ELEMENT character (#PCDATA)>
<!ATTLIST character idcharacter ID #IMPLIED>
<!ELEMENT comment (#PCDATA)>

Use the new DTD file in our XML file. You don’t need to create a new DTD file when you want to update it, it’s just needed to write this example easily.

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie PUBLIC "http://xml-tools.lereskp.fr/examples/movie-1.dtd" "http://xml-tools.lereskp.fr/examples/movie-1.dtd">
<movie idmovie="M1">
 <title>Full Metal Jacket</title>
 <realisator>Stanley Kubrick</realisator>
 <characters>
  <character idcharacter="C1">Matthew Modine</character>
  <character idcharacter="C2">Vincent D'Onofrio</character>
 </characters>
</movie>

To update the file according to the DTD change, we just need to load it without validating the XML and write it. Xmltools detects the missing tags and add it!

>>> xml_filename = 'examples/movie-1.xml'
>>> obj = xmltool.load(xml_filename, validate=False)
>>> obj.write('examples/movie-1-updated.xml')

You can see that the comment tag is added automatically:

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie SYSTEM "http://xml-tools.lereskp.fr/examples/movie-1.dtd">
<movie idmovie="M1">
  <title>Full Metal Jacket</title>
  <realisator>Stanley Kubrick</realisator>
  <characters>
    <character idcharacter="C1">Matthew Modine</character>
    <character idcharacter="C2">Vincent D'Onofrio</character>
  </characters>
  <comment/>
</movie>

Generating a HTML form¶

We will make this example with the same XML file:

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie PUBLIC "http://xml-tools.lereskp.fr/examples/movie.dtd" "http://xml-tools.lereskp.fr/examples/movie.dtd">
<movie idmovie="M1">
 <title>Full Metal Jacket</title>
 <realisator>Stanley Kubrick</realisator>
 <characters>
  <character idcharacter="C1">Matthew Modine</character>
  <character idcharacter="C2">Vincent D'Onofrio</character>
 </characters>
</movie>
Now we will genererate the HTML form:
>>> xml_filename = 'examples/movie.xml'
>>> html = xmltool.generate_form(xml_filename)
>>> # Use etree to have a nicer HTML. In other case we don't need to use it directly.
>>> xml = etree.fromstring(html)
>>> html = etree.tostring(xml.getroottree(), pretty_print=True, encoding='UTF-8', method="html")
>>> open('examples/movie.html', 'w').write(html)

Submitting a HTML form¶

XML tool support you submit WebOb request or dict. We will use a dict for this example.

>>> xml_filename = 'examples/my-movie.xml'
>>> dic = {'_xml_dtd_url': 'http://xml-tools.lereskp.fr/examples/movie.dtd',
...        '_xml_encoding': 'UTF-8',
...        'movie:title:_value': 'My movie title',
...        'movie:realisator:_value': 'My movie realisator',
...        'movie:characters:character__list:1:character:_value': 'My first character',
...        'movie:characters:character__list:2:character:_value': 'My second character',
...       }
>>> obj = xmltool.update(xml_filename, dic)
>>> print obj.title._value
My movie title
>>> print obj.characters.character[0]._value
My first character

The generated XML looks like:

<?xml version='1.0' encoding='UTF-8'?>
<!DOCTYPE movie SYSTEM "http://xml-tools.lereskp.fr/examples/movie.dtd">
<movie>
  <title>My movie title</title>
  <realisator>My movie realisator</realisator>
  <characters>
    <character>My first character</character>
    <character>My second character</character>
  </characters>
</movie>

Note

Before writing the XML file, it is validated according to the DTD file
